Season of Docs ideas

To complement (I guess!) the Google Summer of Code, Google has started a Season of Docs to "Let's bring open source and technical writer communities together, to the benefit of both. "

The TCPUMP group maintains the tcpdump packet decoder, and the libpcap capture library. libpcap is used by wide variety of network diagnostic tools including the very popular wireshark.

TCPDUMP contains two sets of Unix-style manpages: tcpdump(1) and pcap(3pcap). The later of which links to 60-some additional pcap pages. In addition there is a set of link types which describe the kind of information and meta-information that can be found in pcap files, an ancient pcap tutorial. There is an ancient FAQ which is hard to update and hard to add things to.

Some ideas for things to redo:

  • rewrite the manpages to provide better reference, removing tutorial contents, but including "contract" promises
  • create higher-level documentation (in HTML or markdown), with diagrams, to explain how to use libpcap to a programmer new to libpcap
  • illustrate an actual example via an API walkthrough with libpcap. This could involve the tcpsplice code or something new.
  • tuning for better performance of libpcap, advanced details on Linux memory mapped packet interfaces, how to use threads with libpcap
